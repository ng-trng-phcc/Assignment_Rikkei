Giả sử người dùng muốn **xem danh sách sản phẩm**, hãy phân tích yêu cầu này:

- **Client:** Trình duyệt sẽ gửi Request như thế nào? (URL dự kiến, HTTP Method là GET hay POST?).
- **Server:** Ứng dụng Spring Boot sẽ làm gì khi nhận yêu cầu? (Xử lý logic, gọi dữ liệu ở đâu?).
- **Database:** Đóng vai trò gì trong yêu cầu này? (Tạm thời giả lập bằng bộ nhớ RAM).
- **Response:** Server trả về định dạng gì cho Client? (HTML hay JSON?).
---

# Trả lời
## 1. Client (Trình duyệt)
- **URL dự kiến:** `localhost:8080/api/products`
- **HTTP Method:** `GET` (vì đề chỉ đề cập việc người dùng chỉ muốn xem, không thêm/sửa/xóa)

## 2. Server (Spring Boot)
- **Controller** nhận request → gọi Service
- **Service** xử lý logic → gọi Repository
- **Repository** lấy dữ liệu từ RAM (giả lập DB)

## 3. Database (RAM)
- DB đóng vai trò là nơi lưu danh sách sản phẩm tạm thời trong `ArrayList`
- Thực hiện **SELECT** (đọc dữ liệu)

## 4. Response
- **Định dạng:** 
	- chủ yếu là JSON (nếu dùng REST API / `@RestController`)
		- **Ví dụ response dạng JSON:**
		```json
		[
		  {"id": 1, "name": "Laptop", "price": 1500},
		  {"id": 2, "name": "Mouse", "price": 25}
		]
		```
	--- 
	- HTML (nếu dùng `@Controller` + template engine như Thymeleaf)
		- **Ví dụ response dạng HTML (nếu có giao diện):**
		```html
		<ul>
		  <li>Laptop - 1500$</li>
		  <li>Mouse - 25$</li>
		</ul>
		```
